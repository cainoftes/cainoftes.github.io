前言

Markdown 由 John Gruber 于 2004 年创建

在使用 Markdown 格式书写时，文本内容存储在带有 .md 或 .markdown 扩展名的纯文本文件中。然后通过 Markdown App 解析为 HTML 。最后的内容通过 HTML 呈现。

实际上每个 Markdown 应用程序都实现了稍有不同的 Markdown 语法。Markdown 的这些变体通常被称为 flavors（方言）。这就意味着，当你想使用一个 Markdown 应用程序的时候，永远无法确切知道该应用程序到底实现了什么，除非查阅文档。

基本语法

标题

支持 1-6 级标题。

最常用的语法为 #<space><text>。

但是在 <text> 下一行使用 == 来标识一级标题，使用 -- 来标识二级标题

段落

如果要创建段落，需要使用一行空白行来进行分割。

即：段落内部行之间的距离和两段之间的行距离不相同。

在比较高级的编辑器（例如Typora）中，实时渲染的时候可以使用回车来创建新段落。

换行

换行一般使用 HTML 语法来实现。即 <br/>

强调

强调的作用是__加粗__文本或者将文本倾斜。

如果想要加粗文本，在文本之前和之后各自插入**，即**<text>**。当然也可以使用__<text>__，但是兼容性并不好。

如果想要倾斜文本，在文本之前和之后各自插入*，即*<text>*。当然也可以使用_<text>_，但是这种兼容性更加不好（在 Tex 中，下划线会将之后的一个字符转换为下标）。

如果想要加粗的同时倾斜文本，可以在文本之前和之后各自插入***，即***<text>***。同理，还是有___<text>___的写法，但是不建议。

列表

最常使用的列表是有序列表。

语法为 <number>.<space><text>。

需要注意的是，<number> 需要以 1 来开头，但是其余的数字标号不重要，可以随便写，最终效果会进行自动重排。

还有一种列表为无序列表。

语法为 [+-*]<space><text>。

如果要在列表中插入另一种元素但是不改变原本的列表元素顺序，需要使用 Tab 键对插入的元素进行缩进处理。

经过以上方法的组合，也可以实现列表中嵌套列表的结果。

代码

所谓代码就是将文本段进行特殊的显示（具体显示不知道怎么描述）。在代码段前后各插入符号 `。当代码段中有符号`时候，可以使用两个连写的 ` 来表示。

当要使用代码块的时候，需要使用 4 个空格或者一个 Tab 来表示。

实际上，更加常用的是围栏式代码块，见扩展语法。

分隔线

一般在单独的一个空行中使用三个连续的*、- 或者 _ 来形成分隔线的效果。

一般为了兼容性，会在这样的空行前后各插入两个空行。

链接

链接常常用来实现跳转。

最简单的语法如下：[要显示的文字](https://url)。例如[知乎](https://zhihu.com)。

还有一种被称为 Title 的语法，在 url 后空一格，然后使用双引号引用文字。最终的效果是当鼠标悬停在该链接的时候，会显示双引号中的文字。

例如：[知乎](https://zhihu.com "知乎是最好的搜索引擎")

如果方括号和圆括号中的文字相同，即，都是 url ，可以使用<url> 或者 <email>的语法。

除了常用的这种链接之外，还有一种引用链接，将真正的 url 放在文章的另一个地方（例如结尾）。举个例子：

[知乎][1]
[1]: https://zhihu.com "知乎是最好的搜索引擎"

图片

使用图片的方式和使用链接的方式中的一种非常相似，举例如下

![pic][url "可选"]

其中第一个方括号中的文字没有作用。

当然，将图片和上面的链接相结合，最终可以实现点击图片就跳转的功能（图片链接）

引用

引用是改变一段文字的显示效果。

语法为 > <paragraph>

如果要对多个段的文字进行应用，需要在原本的空行开头也插入一个>

还可以在一个引用块中再添加引用。

语法为 >> <paragraph>

嵌入HTML

可以将 HTML 的语法嵌入到 Markdown 中。

对于行级元素，没有过多的限制

低于块级元素，在块元素前后各插入一行空行。

扩展语法

表格

| Syntax      | Description |
| :-----------  | ----------- |
| Header      | Title       |
| Paragraph   | Text        |

围栏式代码块

在代码块的前后各使用三个 ` 或者 ~来表示。

如果要高亮显示代码，可以在行前的` 之后输入具体的编程语言（就看Markdown App 支持性怎么样了）

脚注

脚注引用部分比较简单，使用的语法如下：[^key]。

在定义部分也比较简单，使用的语法如下：[key]: text

标题ID

可以通过在标题之后使用{ID}来表示标题的ID。最大的好处是转换为HTML之后还是会保留ID，可以引用。

这儿我还没有搞清楚

删除线

通过在文本前后添加~~实现删除效果（就是在文本上显示删除线）。

任务列表

任务列表中的任务包括两类，完成的和没有完成的。通过一个 x来体现。需要注意各个字符之间有没有空格的问题。

- [x] 好好学习
- [ ] 天天向上

Emoji

最简单的方法是使用复制粘贴（Emoji本质上还是UTF-8，需要各个环节都使用UTF-8）

另一种方法是使用 Emoji 名和前后添加 :的方法，例如：

:deer:

:deer:

自动网址链接

Markdown 中的 URL 经常会被识别出来，然后当作 URL 来处理，如果不想要自动处理的方式，可以将 URL 当作代码处理，即在 URL 前后添加 `。
