Vim

十月 9, 2022 发布在 default

前言

本文在Windows环境下介绍VIM（GVIM），Neovim，包括安装，使用，配置，插件等部分。

VIM是VI Improved的缩写，起源于VI，诞生于1991年。最初的VIM是指VI Imitation（模拟），最初的那个开发大神只是想复制，后面变成了VI的超集。

Neovim是VIM的一个分支（为什么会用这个词，因为他们都是开源的，开源软件在发展方向上有争议的时候，就会出现多个分支），诞生于2014年。起因是Neovim的发起人对VIM的架构有意见，但是被VIM拒绝了，所以启用了Neovim（Neo-是一个词根，是new在构词的时候的变体词，所以neovim不是no vim的意思）。

VI也是一种文本编辑器，诞生于1976年。其名字来源于ex中的一个命令Visual（视觉的）。在1984年Emacs出现之前，没人任何文本编辑器能在UNIX中和VI竞争。

ex是extended的缩写，是UNIX上用的纯文本编辑器。

在当前，VIM的发展似乎也面临着各种问题，例如VS Code和Sublime Text的出现，在某些层面威胁到了VIM，同时VIM内部也开始分裂，例如Neovim是vim的一个分支，他们对于发展方向之间的差别越来越大了。

安装

在Windows下使用的VIM是GVIM，下载地址如下：download : vim online

当前的VIM版本是9.0（发布于2022-06-28），而上一个版本是8.2。

安装完成之后，使用vim --version可以查看安装的vim的版本。

启动方式分为两种，一种是vim系列，这种在终端模式下启动vim；另一种是gvim，这种使用图像模式启动vim。

个人还是偏向于使用命令行启动vim，即vim系列的命令，毕竟使用vim是为了解决在命令行中编辑困难的问题（Windows有type命令，但是没办法编辑），如果有图像模式的话还不如是用vscode，同时图形模式的UI有点丑。

使用

推荐一个学习VIM的网站 Interactive Vim tutorial - sandbox (openvim.com)，在里边可以练习VIM。

进入

进入vim只需要在CMD中输入vim命令即可，有如下变体。

  • vim：使用vim打开一个空文件（没有保存，需要选择保存的文件名）
  • vim [filename]：使用vim打开[filename]
  • vim +num [filename]：使用vim打开[filename]并且移动到特定的行
  • vim [filename1] [filename2] [filename3] ：同时打开多个文件，如何切换见尾行模式
  • vim -：从标准输入（stdin）读入文件。

三种模式

vim有三种模式：

  1. 命令模式：打开vim之后进入的第一个模式，可以对文件进行一些操作（使用快捷键）
  2. 输入模式：对文件进行编辑，这部分就很正常
  3. 尾行模式：在尾行输入一些命令对文件操作

三种模式之间的转换如下：

  1. 从命令模式到输入模式：
    • i：在当前位置显示出一个 |，也就是在这个位置可以进行输入了
    • I：在当前行的行首（第一个非空白字符）显示出一个 |，然后就可以输入了
    • a：
    • A:
    • o：在当前行的下面插入一个空行，光标停留在下面空行行首
    • O：在当前行的上面插入一个空行，光标停留在上面空行行首
    • s：
    • S：
  2. 从输入模式到命令模式：直接按下Esc键
  3. 从命令模式进入尾行模式：
    • :：按下冒号，然后输入命令
    • /：这种主要是为了查找
  4. 从尾行模式进入命令模式：
    • Enter：输入命令之后回车，命令执行，并且返回命令模式
    • Esc：不执行命令退回命令模式
  5. 进入VIM：参照上面的进入命令
  6. 退出VIM：在命令模式下输入两个大写的ZZ。

需要注意以下几件事，

  • 输入模式和尾行模式之间的转换必须通过命令模式
  • 尾行模式也可以直接退出，也就是q命令，具体参照下面
  • 命令的执行在尾行模式，而命令模式则主要是快捷键操作

命令模式

  1. 光标移动

    • 移动到行首：^（其实^很多时候都代表开始，例如正则表达式）
    • 移动到行尾：$（其实$很多时候都代表开始，例如正则表达式）
    • 移动到首行：gg
    • 移动到末行：G
    • 上一页：PgUp（下一页同理）
    • 光标移动到第 n 行：nG（例如2G会移动到第二行）
    • 光标向下移动 n 行：n↓（例如2↓会将光标移动到当前行的下两行，向上、左、右同理）

  2. 复制剪贴

    • 复制当前行：yy

    • 粘贴：p，需要注意的是，这只能用于在vim里边复制的东西，如果是从别处复制来的，还是要使用系统带的粘贴命令ctrl + shift + V

      为什么会这样？不知道，具体的实现可能和什么寄存器相关

    • 复制当前行及其下 n - 1 行：nyy

    • 可视化复制：使用 v 就可以高亮显示选中的文字，然后使用 y复制

    • 剪切当前行：dd（包括换行符，即下一行上移）

    • 剪切当前行及其下 n - 1行：ndd

    • 可视化剪切：使用 v 就可以高亮显示选中的文字，然后使用 d剪切

  3. 撤销恢复

    • 撤销：u
    • 恢复：ctrl + r

尾行模式

  1. 保存更改：:w，或者:w [path]
  2. 退出文件：:q，可以和w组合使用
  3. 强制执行：:!，可以和q组合使用
  4. 查找关键词：/word，会高亮显示，然后使用N/n切换上下
  5. 显示行号：:set nu
  6. 取消行号显示：:set nonu
  7. 显示当前打开的所以文件：:files
  8. 切换到打开的其它文件：:open [filename]

编辑模式

没啥好说的，就跟平常的notepad一样编辑就可以，VIM主要复杂在前两种模式上。

配置

配置文件如下，可以使用vim --version查看

  • 系统 vimrc 文件: “$VIM\vimrc”
  • 用户 vimrc 文件: “$HOME_vimrc”
  • 用户 exrc 文件: “$HOME_exrc”
  • defaults 文件: “$VIMRUNTIME\defaults.vim”

其中以$开头的都指向一个具体的路径，以$VIM为例，可以通过在尾行模式中输入:echo $VIM查看

vimrc

可以参考此处 [Vim 配置文件 vimrc 入门简介 - MyBatis中文官网](http://www.mybatis.cn/archives/1303.html#:~:text=.vimrc是Vim的配置文件，Vim编辑器相关的所有功能开关都可以通过.vimrc文件进行设置。 备注：文件名中的 “rc”,是出自 “run commands”。)

vimrc是vim的配置文件，其中的rc是run commands的简写。系统vimrc放在vim安装目录下，用户vimrc放在home目录下。

由于有用户和系统两个配置文件，当他们发生冲突的时候，会取用户vimrc中的配置。

vimrc里边以 “ 开头的行，都是注释行，没有任何用处

以下是我自己添加的

set number
set cursorline
set showmatch
set tabstop=4
set ruler
" 文件在vim之外修改过，自动读入 "
set autoread
" 配色方案"
colorscheme desert
