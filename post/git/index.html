Git

2022-10-1 发布在 default

Git教程 - 廖雪峰的官方网站 (liaoxuefeng.com)

开篇

  • 要解决什么问题

    版本控制，想一下一个小组作业，每个人完成一部分代码，每个人的代码是相互联系的，你的代码依赖别人的代码，你的代码被别人的代码依赖着，要是你一修改代码，立马出现一堆 Error。如果没有小组作业的经历，想一下写过的Word文档，每个文档经历大的修改的时候，总是要记录下之前一个版本（如果你说你不想记怎么办？….. 没办法，文件丢失一次之后就会想记录了），所以命名的时候会出现1，2，3，4，但是你总是想不起来当前的各种版本之间的差别……没有版本控制就是一场灾难

  • 怎么出来的？

    天地初开的混沌时期，Linus创立了Linux，一呼百应，许多大神追随Linus的脚步为Linux贡献代码。这些代码由Linus手工合并，因为Linus不喜欢免费的CVC和SVN，至于为什么不喜欢，因为这些都是集中式的版本控制。直到Linux大到Linus无法手工维持的时候……

    Linus开始采用BitKeeper，这是一个商业系统，但是Linus和Linux在江湖上实在德高望重，为江湖开辟了一条新的修炼体系，提携了无数后辈，所以BitKeeper授予Linux社区免费使用这个系统。

    但是Linux社区的人自诩为有才，妄图攻陷（破解）BitKeeper协议，被东家发现了，所以东家收回了免费使用权。

    Linus大神当时怎么想的已经不可知了，但是一个月之后，Git系统横空出世（在现在Git是什么地位大家都知道了），所以Git和Linux携手发展。

    3年之后的2008年，Github网站上线，一时间成为最大的同性交友社区。

安装（Windows）

下载地址：https://git-scm.com/downloads

安装完成之后可以在安装目录中找到两个可执行文件

  • git-bash.exe：Linux风格
  • git-cmd.exe：Windows风格

也可以将安装目录下的CMD文件夹配置到环境变量中，之后在CMD中和Powershell中可以直接使用 git 命令，更加方便。

创建版本库

  • git init：将当前文件夹变成使用git管理的目录
  • git init [Path]：将Path文件夹变成使用git管理的目录

以上两个命令都不限制文件夹是否是空的，如果不是空的也删除已经存在的文件。

执行完上面两个命令之后在文件夹中会出现一个新的文件夹（隐藏起来的）.git，里边的文件如下

  • git add File
  • git add Path
  • git add .

以上三个命令就是将文件添加到仓库

  • git commit -m "随便写些什么"

将前边add的文件提交到仓库

  • git status：显示当前的状态
  • git diff file ：显示文件的改动
  • git log：显示日志

分支管理

git默认的分支是master，2020.10.1之后github的主分支从master变成了main，理由是master and slave 种族歧视 ……..

  • git branch dev：创建新分支dev

    • -b：创建并切换
    • -d：删除

  • git checkout dev：切换到dev分支

  • git switch dev：切换到dev分支（新来的命令，更加科学）

    • -c：创建并切换

  • git branch：列出所有分支，并在当前分支前加一个*号

  • git merge [repo]：将repo分支合并到当前的分支

    通常使用的是Fast forward模式，

远程协作

远程仓库的默认名字是origin，使用git remote -v 命令之后可以查看。

将本地代码推送到远程仓库，实际上是将本地的一个分支推送到远程的一个分支

  • git push origin main
