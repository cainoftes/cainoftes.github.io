前言

  接触的第一门编程语言是C。学习C的时候磕磕绊绊，有很多地方不够精通。后面又解除了一堆的语言，却总是摆脱不了C对我的影响。

  程序有三种结构：顺序、选择、循环。有了这三种结构之后，理论上就可以表达所有的语义（对C来说，又提供了goto语句，所以可以不需要循环，但是这样会破坏它本身的结构化，结构化编程有它自己的优势。）。这就是为什么C的地位这么高的一部分原因——它刚好卡在了一个精妙的边界上，删减一部分之后，C会变成低级语言，增加一部分之后，C又会变得臃肿。

  但是正是基于以上原因，后面学习的所有语言，我都当成了C来使用，不管是Java，Python，C#，还是C++。更确切来说，是把他们当成C + API来使用，直到前几天对比了十几份实现同一功能的Python程序，才发现我真的是在写C，满满的方言味。同时也第一次见识了Python的华丽和强大，我写的复杂逻辑，Python本身就可以用一条语句直接调用！

  所以我来改正自己狭隘的思想来了，做一份索引，方便以后的查找。是为序！

内置序列类型

容器序列

序列中的每一个元素都是对象的引用。这是实现容器序列可以存放不同类型的数据的基础。

list tuple collections.deque

扁平序列

序列中的每一个元素都是真正的值，所以它里边只能存放同一类、基础类型的元素，更详细一点，扁平序列对应的是一段连续的空间（内存空间）。

str bytes bytearray memoryview array.array

不可变序列Sequence

tuple str bytes

更详细地，可以说Mutable继承了Sequence。

最重要最基础的List

列表推导list comprehension(listcomps)

  1. 最简单的示例

    symbols = '$¢£¥€¤' 
codes = [ord(symbol) for symbol in symbols] 
print(codes)
# [36, 162, 163, 165, 8364, 164]
    • 一般来说，listcomps只在创建列表的时候使用，否则可以使用生成器表达式！
    • listcomps建议使用的场景：创建列表且代码简短。
    • 值得一提的是，列表推导是一个局部作用域，如上中的symbol不会上下文的symbol产生影响（Python 3.x来出现，Pythoon 2.x 会改写）
    • 列表推导并不是lambda表达式。

  2. 对元素进行过滤

    # 可以对列表使用过滤
beyond_ascii = [ord(s) for s in symbols if ord(s) > 127]
print(beyond_ascii)
# [162, 163, 165, 8364, 164]

    可以是用另一种方式过滤

    beyond_ascii = list(filter(lambda c: c > 127, map(ord, symbols)))
print(beyond_ascii)
# [162, 163, 165, 8364, 164]

  3. 多维的listcomps（笛卡尔积）

    colors = ['black', 'white']
sizes = ['S', 'M', 'L']
tshirts = [(color, size) for color in colors for size in sizes]
# [('black', 'S'), ('black', 'M'), ('black', 'L'), ('white', 'S'), ('white', 'M'), ('white', 'L')]
# 过滤写法
tshirts = [(color, size) for color in colors if color == 'black' for size in sizes if size == 'S']

生成器表达式

使用listcomps也可以创建诸如元组之类的数据类型，但是这种方式本质上是创建一个列表，然后调用构造函数，所以性能并不怎么好！

  1. 简单的生成器表达式

    symbols = '$¢£¥€¤' 
tuple(ord(symbol) for symbol in symbols)
    • 可以看出，生成器表达式和列表推导及其相似，只是把[]更换为了()
    • 生成器表达式逐个产出元素（tips：可以用在for循环中，节省内存空间）

元组

元组是"不可变列表"

  1. 元组的拆包

    record = ('Tokyo', 2003, 32450, 0.66, 8014)
# 其中_就是一个占位符
city, year, _, chg, area = record
print('%s/%s/%s/%s/%s' % record)
# Tokyo/2003/32450/0.66/8014

    其实元组的拆包可以用在任何可迭代元素上

    # 优雅！永不过时
a,b = b,a

    可以将一个元组显式拆开

    t = (20, 8)
divmod(*t)

    可以使用*来处理剩余的元素

    a, b, *rest = range(5)
print(rest)
# [2, 3, 4]
# 竟然可以出现在任意的位置上
a, *rest, b = range(5)
print(rest)
# [1, 2, 3]

  2. 具名元组

    from collections import namedtuple
City = namedtuple('City', 'name country population coordinates')
tokyo = City('Tokyo', 'JP', 36.993, (35.689722, 139.691667))
print(tokyo)
# City(name='Tokyo', country='JP', population=36.993, coordinates=(35.689722, 139.691667))
print(tokyo[1])
# JP
print(tokyo.name)
# Tokyo
print(City._fields)
# ('name', 'country', 'population', 'coordinates')

字典

你知道吗？创建字典至少有六种姿势

a = dict(one = 1, two = 2, three = 3)
b = {'one': 1, 'two' : 2, 'three' : 3}
c = dict(zip(['one', 'two', 'three'], [1,2,3]))
d = dict([('two', 2), ('one', 1), ('three', 3)])
e = dict({'three':3, 'one':1, 'two':2})

  1. 字典推导

    DIAL_CODES = [ (86, 'China'), (91, 'India'),  (1, 'United States')]
country_code = {country:code for country , code in DIAL_CODES}

  2. 在字典中查找一个键对应的值

    key = 86
country_code[key]
# 如果上面的语句找不到key，会抛出一个异常（“快速失败”的哲学？？？）
# 可以使用下面的方法来更快处理
country_code.get(key, default)
# 上面这段程序的意思是首先寻找，如果找到键就返回值，不然就返回defaUlt的值

集合

集合？不就是用来去重的吗？？？

l = ['spam', 'spam', 'eggs', 'spam']
a = set(l)
print(a)
# a = set(l)
l = list(a)
print(l)
# ['spam', 'eggs']
## 下面的写法更加简洁
l = list(set(l))
print(l)
# ['spam', 'eggs']

  1. 集合的运算符

    a = set()
b = set()
a | b
a & b
a - b
a in b
a <= b

  2. 集合推导

    from unicodedata import name
{chr(i) for i in range(32, 256) if 'SIGN' in name(chr(i),'')}
