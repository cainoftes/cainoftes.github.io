Hugo搭配Github Pages发布静态博客

十月 1, 2022 发布在 default

学会，然后完成！而不是，完成，再学。

工具介绍

  • 站点托管工具：Github Pages

    这是github为用户提供的一个免费功能，用户将自己的Html(CSS, JS)代码上传到Github Pages，Github Pages将这些通过网站的方式显示出来。相当于提供了一个只能发布网页的服务器。

    官方文档：github-pages

  • 静态网站生成器：Hugo

    什么是“静态”：静态是指用户和网页不能交互，不能动态改变显示的HTML的页面内容（评论系统除外）

    Hugo 使用Go语言开发，是一个静态网站生成器。更简单理解，它将Markdown写成的文档组装为HTML和CSS代码。 之前看到一句话，颇有道理：“这是一个笔记软件比笔记还多的时代”。夸张一下放到博客系统也成立：“这是一个博客系统比博客还多的时代”。除了Github Pages之外还有Gitee Pages……，除了Hugo 还有 Hexo，Jekyll……甚至还有写博客的软件，例如Gridea。

以上举例中提到的工具，我都使用过，反而发现自己背离了初心：我在为菜刀雕花。菜刀只是用来切菜的，其上到底用什么花纹似乎并无关系。

所以最后选择了Github pages和Hugo来搭建自己的博客系统，在此立下Flags，可预见的时间内，不会更换其它的博客系统。

Hugo生成静态博客

Windows下的Hugo下载

官方地址：https://gohugo.io/getting-started/installing/

根据以上文档，详细说明以下两个安装方式

安装包安装

由于Hugo是一个开源项目，其代码托管在Github中。Github也提供了软件的Release版本，地址为Releases · gohugoio/hugo (github.com)。

（似乎Windows下暂时不能通过安装包安装）

包管理器安装

包管理器也是一个软件，它可以管理其它的软件（并不是所有的软件都可以被其管理，总有一些漏网之鱼只能用安装包管理）。这里的管理指：安装，卸载，修改等。

Scoop 安装教程Windows下Scoop安装、配置与使用 - 腾讯云开发者社区-腾讯云 (tencent.com)

使用Scoop管理器安装，使用以下命令

scoop install hugo

Hugo使用

Hugo中文文档 (gohugo.org)

  1. 新建一个Hugo网站存放文件夹

    hugo new site hugo-folder：在当前目录下新建了hugo-folder文件夹，文件夹中被hugo初始化，里边有hugo运行需要的文件和文件夹。

    下文中的 . 文件夹代表hugo-folder文件夹

  2. 选一个主题，下载到 hugo-folder/themes文件夹下

    主题可以在Complete List | Hugo Themes (gohugo.io)中选择，推荐使用Tranquilpeak 主题（可以跟着教程配置一遍，走完整个流程再选择自己喜欢的主题）

    跳转到hugo-folder/themes文件夹下，使用以下命令

    git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git tranquilpeak：将github的hugo-tranquilpeak-theme仓库下载到themes文件夹下的tranquilpeak文件夹中

  3. 修改主文件夹下的config.toml文件，这个文件本质是一个文本文件（配置文件，作用类似于.json .yaml等格式），可以使用各种文本编辑器打开

    使用 ./themes/tranquilpeak/exampleSite/config.toml文件替换 config.toml文件。

    这是一个模板文件，已经配置好了，下文会详细介绍如何修改

  4. 新建博客

    在 . 目录下使用 hugo new post/my-first-post.md命令可以生成名为 my-first-post.md的Markdown文件。

    打开之后在里边添加文章（头部不要修改，在已经存在的部分下面添加，头部的作用和格式会在后文说明）

  5. 生成网站并查看

    在 . 目录下运行 hugo命令，会生成./public文件夹，文件夹下就是HTML和CSS文件。

    在 . 目录下运行 hugo server命令，该命令会在本地主机（自己的电脑）中生成一个网站，可以在浏览器中通过 http://localhost:1313/ 访问。事实上，在运行完该命令的结果中就要如何查看的提示。

    如果有相关的开发经验，就知道这种访问方式很常见。

    这一步应该生成一个静态博客网站，其中可以看到在第四步中写的文字。

部署到Github Pages

建立仓库

新建立一个仓库，仓库名字命名为 [name].github.io

如果你的github用户名为 zhangsan，那仓库名为zhangsan.github.io

如果你的github用户名为 lisi，那仓库名为lisi.github.io

将本地文件托管到github

生成的博客位于 ./public文件夹下，所以以下命令在 ./public 文件夹下进行

  • git init：将当前文件夹初始化为git仓库

  • git add .：将当前所有文件加入git暂存区

  • git commit -m "初次提交"：提交暂存区文件

  • git remote add origin [info]：关联远程库文件

    [info]需要修改为自己的地址，可以在github仓库中的Code（那个绿色的按钮）下复制ssh信息

  • git branch -M main：将当前分支从master修改为main

    因为git默认的仓库是master，而github默认的仓库是main（原本也是master，2020年因为一些原因修改为了main），会导致不一致。

  • git push origin main：推送

查看

可以通过 zhangsan.github.io （zhangsan修改为自己的github名）查看自己的博客，需要注意的是，从推送到github再到更新，可能有一段时间（1min这个样子）
