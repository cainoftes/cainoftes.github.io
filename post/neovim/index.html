前言

关于 vim 的基本操作和 vim 的配置可以参考前一篇文章 ： Vim (cainoftes.github.io)

本篇将会详细说明 Neovim 在 Windows 下的配置和插件。

配置

基础篇

VIM 的配置文件为 init.vim，Neovim 也支持 init.vim 配置，但是同时，Neovim 也使用 init.lua 配置，即使用 lua 语言，这也就是所说的 Neovim 是 VIM 的超集。

另外，init.vim 的功能比 init.lua 稍微强一点， 有部分功能暂时还没办法使用 init.lua 配置。*.vim 文件优先于 *.lua 文件调用。

在 Linux 下，用户配置文件为 ~/.config/nvim/init.vim；在 Windows 下，用户配置文件为 ~/AppData/Local/nvim/init.vim；

（这也就是我折腾了好久的地方，以为 Windows 下和 Linux 下一样。大部分的博客都不会说明这件事情，可能它们默认都在 Linux 下工作）

Vim命名空间

  • vim.{o/wo/bo}: vim 选项，例如 :lua vim.wo.number = true 等价于 :set number
  • vim.g：全局变量，相当于 set g:variable = value
  • vim.env: 访问和配置环境变量
  • vim.fn: 访问 vim 运行时函数，例如 :lua print(vim.fn.getenv('HOME')) 等价于 :echo getenv('HOME') 写法
  • vim.api: 提供了一些 api 代替 vim 命令，例如不严格来说，vim 中的 map 对应 vim.api.nvim_set_keymap
  • vim.cmd: 在 lua 中执行 vimscript，例如 :lua vim.cmd('echo getenv("HOME")')
  • vim.lsp: 控制内置 LSP 客户端的模块，例如：:lua vim.lsp.buf.formatting()
  • vim.inspect: 可以把 vim 运行时对象转成可以打印的 lua 对象格式，例如 lua print(vim.inspect(vim.opt.completeopt))

init.vim

这是 neovim 启动时直接调用的文件，所有的用户配置都要放在该文件中或者在该文件中调用。

调用方式：使用 lua require('basic')来调用 lua 目录下的 basic.lua 文件。

LSP

LSP 是指语言服务协议，由微软提出。LSP 将编程语言分为了 Language Server 和 Language Client 两个部分。Server 就相当于后端，提供代码补全、查找引用、跳转到定义等功能。
