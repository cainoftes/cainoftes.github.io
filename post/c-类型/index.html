字段Field

Field是为一个对象或者一个类型存储数据的东西。本身也是一种变量，叫“成员变量”。

为一个类型存储数据？也就是静态，和类型直接关联。

例如说：有一个Student类，类中有age，有score。这是每一个学生都不同的，是和学生对象相关联的。Student还有AverageAge，AverageScore，但是这个是和具体的学生无关的，是整体的特征，使用Static类型。

需要注意的是：

  • 字段的名字需要使用名词，不能使用动词或者动词短语。
  • 字段的声明不是语句，后者只能出现在函数体中，字段只能出现在类体中。
  • 对于C/C++来说，字段（实际在C/C++并没有显示命名）的初始化只能出现在构造函数中，但是对C#来说，可以直接放在后面（实际还是在构造函数中初始化）。而且建议放在后面，这样不会随着构造函数而改变。如果没有显示赋值，就是default的值。
  • 有一种东西叫静态构造器，将public换为static
  • 有种变量叫readonly。例如说学生的ID就是一个readonly，这个readonly放在构造函数中。

属性Property

这和Field不是一样的吗？

  • Field命名更偏向于存储，而Property更偏向
  • 属性用来对外暴露数据，一般不会使用Field来对外暴露数据
  • 属性可以是计算出来的
// 以下是C++/Java中的写法。C#狂笑，骄傲！
Class Student{
	private int age;
	public int GetAge(){
		return this.age;
	}
	public int SetAge(int value){
		if(value >= 0 && value <= 120) {
			this.age = value;
		}
		else{
			// 抛出异常是好事，我们就拥有了Debug的机会。
			throw new Exception("Age value has error!");
		}
	}
}
// 以下是C#写法
Class Student{
    private int age;
    public int Age{
        get{
            return this.age;
        }
        // 你甚至还可以直接把这个前边放一个private
        set{
            // value 是上下文关键字，在这儿就代表用户传进来的关键值
            // 一旦离开了这儿，value就不是关键字了。
            if(value >= 0 && value <= 120) {
				this.age = value;
            }
            else{
				throw new Exception("Age value has error!");
            }
        }
    }
}
// C#还有一种属性的声明方法，但是这种方法不会保护数据，只是用于对外展示数据用的
Class Student{
    public int Age{get;set;};
}

语法糖：一种简单的语句，但是背后隐藏了大量的工作。

而属性，就是一种语法糖。背后还是生成了两个函数。

  • 属性可以省略get/set，但是一般不会出现set隐藏get

怎么理解属性，举个例子，在上面这个类中你可以放一个canWork的熟悉，然后通过年龄判断他到底能不能工作，例如

// 常用来设置的版本
Class Student{
    private int age;
    public int Age{
        get{
            return this.age;
        }
        // 你甚至还可以直接把这个前边放一个private
        set{
            // value 是上下文关键字，在这儿就代表用户传进来的关键值
            // 一旦离开了这儿，value就不是关键字了。
            if(value >= 0 && value <= 120) {
				this.age = value;
            }
            else{
				throw new Exception("Age value has error!");
            }
        }
    }
    public canWork{
        get{
			if(Age > 16) {
				return true;
            }
            else {
                return false;
            }
        }
    }
}

// 常用来查询的版本
Class Student{
    private int age;

常量

常量属于类型。属于的意思是说，一个常量隶属于一个类/结构体……。

而readonly本质来说，修饰的是一个变量，在内存中的存储是和变量一样的。

编译的时候，会直接将常量的那个名字替换为一个值。

还需要注意的是，const修饰的只能是普通类型，如果是class就不行。

可以使用static readonly来修改。
